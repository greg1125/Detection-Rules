## **Brute Force Authentication Detection (SSH & RDP)**

This detection focuses on identifying brute force authentication attempts against both SSH and RDP services. Brute force attacks occur when an attacker repeatedly attempts to authenticate using multiple username and password combinations in order to gain unauthorized access.

In this lab environment, brute force activity was simulated using the Kali Linux tool `ncrack`, which is commonly used to perform password attacks against network services such as SSH and RDP.

These attacks generate repeated failed authentication attempts within a short period of time, creating a detectable pattern that differs significantly from normal user behavior.

---

## **Detection Rule**

This detection is implemented using threshold-based KQL rules within Kibana to identify repeated failed authentication attempts from the same source.

### **SSH Brute Force Rule**

```kql
system.auth.ssh.event:"Failed" and user.name:*
```
Threshold condition: Grouped by: source.ip, user.name Trigger when attempts ≥ 5 within 5 minutes RDP Brute Force Rule event.code:4625 and winlog.event_data.LogonType:"10"
---

## **Detection Logic**

This detection is designed using a behavior-based threshold approach.

Failed authentication attempts are normal in small amounts, but a high volume of failures within a short time window is indicative of brute force activity.

For SSH, failed login attempts are captured through system authentication logs, where repeated failures from the same source IP indicate password guessing attempts.

For RDP, failed login attempts are recorded as Event ID 4625 with Logon Type 10, which represents remote interactive logons commonly used for Remote Desktop sessions.

By aggregating failed authentication attempts and applying a threshold, the detection identifies abnormal login behavior that is unlikely to occur during legitimate use.

## **MITRE Mapping**

This activity maps to the following MITRE ATT&CK technique:

T1110 – Brute Force

This technique describes attempts to gain access to accounts through repeated login attempts using different credential combinations.

## **Attack Simulation (Kali - ncrack)**

The brute force activity was generated using Kali Linux and the `ncrack` tool.

### **SSH Attack**

```bash
ncrack -vv --user <username> -P greg-wordlist.txt ssh://<target-ip>
```
This command performs a brute force attack against an SSH service by attempting multiple password combinations for a given user account.

The repeated failed login attempts generated by this attack are captured in system authentication logs and used to trigger the detection rule.

## **Key Insight**

Brute force attacks generate a high volume of failed authentication attempts that can be reliably detected using threshold-based rules.

Unlike more stealthy attack techniques, brute force activity is noisy and produces a clear pattern of repeated failures, making it well-suited for detection through aggregation and rate-based analysis.

---

## **Operational Context**

This activity represents a likely credential attack against exposed authentication services.

While occasional failed logins are expected, repeated failures from a single source within a short timeframe strongly indicate malicious intent.

If successful, brute force attacks can lead to account compromise, unauthorized access, and further lateral movement within the environment.

Analysts should prioritize identifying the source of the attack and determining whether any login attempts were successful.
